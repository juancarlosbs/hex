# Architecture — Hex

> How Hex is structured: layers, responsibilities, contracts between modules, and data
> flows. Rule for the agent: respect the boundaries on this page. Business logic in
> the Rust domain; React is presentation only. In conflict, this doc + the ADRs win over code.
> See `docs/decisions.md` for the *why* behind each decision; this page covers the *how*.

---

## 1. Principle

**Hexagonal (ports & adapters) in Rust · pure presentation in React · typed contract between the two.**

- The Rust **domain** (request model, `SchemaNode`, envelope, timing) is **pure, no I/O**. Unit-testable, and the demonstrable core of the project (ADR-003).
- **Adapters** do I/O: HTTP engine (instrumented hyper), WSDL/XSD parser, file persistence, and the **commands** layer (Tauri IPC).
- The **frontend** is presentation + interaction only. It never speaks HTTP directly (ADR-004); it calls Rust via `invoke` with **generated** types (tauri-specta, ADR-007). Changing a command signature breaks the frontend at compile-time.

**Golden boundary rule**: validation and business logic are **authoritative in Rust**.
React may validate for UX (immediate feedback), but the domain never trusts what came from the form.

## 2. Folder structure

### Rust — `src-tauri/src/`
```
domain/            # PURE, no I/O — the testable heart
  request.rs        # Protocol{Rest,Soap}, HttpRequest, SoapRequest, RequestModel
  schema.rs         # SchemaNode (central piece), XsdType, Occurs{min,max}, Choice, EnumValues
  envelope.rs       # SOAP envelope model (namespaces, header, body)
  response.rs       # ResponseData, TimingBreakdown, ResponseNode (tree), SoapFault
  error.rs          # domain error types (thiserror)
engine/            # transport adapter (I/O)
  client.rs         # orchestrates request phases -> ResponseData + TimingBreakdown
  connector.rs      # custom Connect: stamps Instant at DNS/TCP/TLS (the waterfall)
  serialize.rs      # SchemaNode + form data -> SOAP envelope (correct namespaces)
wsdl/              # parsing pipeline (input port)
  parse.rs          # WSDL traversal (roxmltree): service/binding/portType/operations
  resolve.rs        # transitive resolution of import/include (fetch of external schemas)
  xsd.rs            # XSD -> SchemaNode (ADR-010 subset)
persistence/       # collections/history (serde + fs)
  collection.rs     # 1 file per request (YAML/TOML); tree <-> disk
  history.rs        # execution history (optional SQLite; separate from collections)
commands/          # #[tauri::command] THIN handlers — validate and delegate, no logic
  mod.rs            # send_request, import_wsdl, create_workspace, save_request, ...
lib.rs             # Builder, tauri-specta export, plugin registration
```

### React — `src/`
```
bindings.ts          # GENERATED by tauri-specta — DO NOT edit manually
lib/
  api.ts             # thin wrappers over bindings (sendRequest, importWsdl, ...)
store/               # zustand slices
  workspace.ts        # collections, tabs, active request
  environment.ts      # environments and variables
  settings.ts
features/
  schema-form/        # RECURSIVE SchemaNode -> form renderer (UI core)
  response-tree/      # recursive tree + copy-leaf
  timing/             # waterfall
  wsdl-import/         # modal + import flow (loading/resolution error)
components/
  titlebar/  sidebar/  request/  response/  ui/   # ui/ = shadcn
hooks/
styles/tokens.css    # shadcn + semantic variables (single source of color)
```

## 3. Layers and contracts

Dependency flow (only points inward; nothing in the domain knows about the outside):

```
commands  -->  engine / wsdl / persistence  -->  domain
   ^                                              (depends on nobody)
   |
  IPC (tauri-specta) --> React (lib/api.ts --> features/components)
```

Contracts per layer:

- **`domain/`** — only types and pure functions. Zero `tokio`, `reqwest`, `std::fs`, `tauri`. If it needs I/O, it's not domain.
- **`engine/`** — receives a `RequestModel`/`Envelope` from the domain, does the network call, returns `ResponseData`. The only place that speaks hyper/rustls.
- **`wsdl/`** — receives URL/bytes, returns `Vec<Operation>` where each `Operation` carries a `SchemaNode`. The only place that resolves external schemas.
- **`persistence/`** — serializes/deserializes collections and history. The only place that touches the data filesystem.
- **`commands/`** — translates IPC <-> domain. Validates input, calls the right layer, maps errors to responses. **No business logic.**
- **React** — consumes `bindings.ts` via `lib/api.ts`. Components never call `invoke` with a bare string.

## 4. Applied patterns

- **Ports & adapters** (ADR-003) — domain isolated from hyper/fs/IPC.
- **Pipeline** in WSDL — `parse -> resolve(imports) -> xsd -> SchemaNode`, each stage a pure function testable in isolation.
- **Connector decorator** (ADR-005) — a `Connect` that wraps the transport and stamps timestamps per phase. This is what generates the real waterfall.
- **Typed command pattern** (ADR-007) — thin commands; specta generates the TS contract.
- **Recursive component** (frontend) — `SchemaForm` and `ResponseTree` follow the same pattern: render a node and map over children. `xs:choice` -> branch selector; `maxOccurs>1` -> repeatable group; `minOccurs=0` -> collapsed optional.
- **Typestate (optional, bonus)** — `Request<Draft> -> Request<Validated> -> Request<Sent>`: forces required fields to be validated in the type before sending.

## 5. Data flows

### Send request (flow F3 from product.md)
```
SchemaForm collects data
  -> lib/api.sendRequest(requestModel)          // typed invoke (bindings.ts)
       -> command::send_request                 // thin IPC adapter
            - validates (authoritative)
            - engine::serialize (SOAP: SchemaNode+data -> envelope) | (REST: builds HttpRequest)
            - engine::client with instrumented connector
                 -> { status, headers, body, TimingBreakdown }  |  SoapFault
  -> UI: ResponseTree (copy-leaf) + Timing waterfall
```

### Import WSDL (flow F2 from product.md)
```
lib/api.importWsdl(url|bytes)
  -> command::import_wsdl
       -> wsdl::parse
            -> wsdl::resolve(imports/includes)   // can FAIL: 404/timeout/invalid
                 -> wsdl::xsd -> SchemaNode per operation
       -> Vec<Operation{ name, endpoint, soapAction, input: SchemaNode }>
  -> UI: builds the tree in the sidebar; clicking an operation loads SchemaNode -> SchemaForm
```
`resolve` errors surface as typed errors and become clear messages in the modal (which schema, why). Never silently import partial results.

### Streaming large responses
Single IPC return serves the common case. For large responses, the `engine` emits **Tauri events** (chunks/progress) instead of a single return — the frontend subscribes to the event and fills the tree progressively. (v2 if not needed in MVP.)

## 6. Where things CANNOT be (anti-patterns to reject)

- Business logic in `commands/` or React components.
- `fetch`/XHR in the frontend for external APIs (ADR-004).
- `std::fs`/network in `domain/`.
- Hardcoded color in a component (only `styles/tokens.css`).
- `invoke("string", ...)` outside of `lib/api.ts`.
- Editing `bindings.ts` manually (it's generated).

## 7. Milestones (implementation order)

Goal: get the **hero flow** running end-to-end as early as possible.

1. **Scaffold** — Tauri v2 + Vite/React19/TS; tauri-specta generating `bindings.ts`; shadcn + tokens.
2. **Shell + titlebar** (decorum) + three-pane (`react-resizable-panels`) with persisted state.
3. **Minimal REST engine** (reqwest shortcut, ADR-005) -> typed `send_request` command -> `ResponseTree`.
4. **Instrumented connector** (hyper + hickory + tokio-rustls) -> `TimingBreakdown` -> waterfall.
5. **Import WSDL** — `wsdl::parse` + `resolve` (with handled errors) -> operations tree in sidebar.
6. **Recursive `SchemaForm`** (required/optional/enum/array/choice) + `engine::serialize` for envelope.
7. **Form ⇄ XML** (CodeMirror) + SOAP Fault rendered as error.
8. **Polish** — Cmd-K (cmdk), XPath/JSONPath filter, virtualization, themes, refined copy-leaf.

Steps 2->6 are the critical path. Library/version details in `docs/stack.md`;
domain types in `docs/domain-model.md`; SOAP engine in `docs/soap-engine.md`.
